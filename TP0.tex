\exercice{- Paramétrer son navigateur internet}
Procédure :
\begin{enumerate}
\item Ouvrir un navigateur internet (ex : \texttt{firefox})
\item Cliquer sur le menu \textbf{Edition} puis choisir \textbf{Préférences}.
\item Choisir l'onglet \textbf{Avancé} puis \textbf{Réseau}.
\item Choisir \textbf{adresse de configuration automatique du proxy}.
\item Taper l'adresse: \verb|http://mailhost.polytech.upmc.fr/prox.pac |
\item Ouvrir à nouveau votre navigateur et vérifier que vous avez accès à Internet.
\end{enumerate}

\exercice{- Répertoire courant}
Procédure : 
\begin{enumerate}
\item Ouvrir un terminal (fenêtre non graphique dans laquelle vous pouvez entrer des commandes qui seront interprétés par un programme appelé "Shell")
\item Pour afficher le nom du répertoire courant (si vous venez d'ouvrir un shell, il s'agit du répertoire d'accueil), taper la commande \texttt{pwd} (pour s'en souvenir : \textbf{p}rint \textbf{w}orking \textbf{d}irectory).
\item Pour afficher le contenu du répertoire courant, taper la commande \texttt{ls} (pour s'en souvenir : \textbf{l}i\textbf{s}t).
\item Pour retourner au répertoire d'accueil, taper la commande \texttt{cd} (pour s'en ouvenir : \textbf{c}hange \textbf{d}irectory).
\item Pour créer un répertoire : \texttt{mkdir} \verb|temp| (par exemple) (pour s'en souvenir : \textbf{m}a\textbf{k}e \textbf{dir}ectory).
\item Pour changer de répertoire courant/travail:

\texttt{cd temp} (chemin relatif, accède au sous-repertoire \texttt{temp} du répertoire courant/travail)

\texttt{cd \textasciitilde/temp} (chemin absolu, accède au répertoire désigné par un chemin unique depuis la racine)
\item Pour aller dans le répertoire parent (chemin relatif) : \texttt{cd ..}
\end{enumerate}

\exercice{- Manipulation de fichiers texte}
\begin{enumerate}
\item À partir de votre répertoire d'accueil (rappel : il est désigné par le raccourci \verb|~|), créer un répertoire \verb|info|
\item Aller dans le répertoire \verb|info| et y créer un répertoire \verb|TP0|.
\ifdefined\matlab
\item A l'aide d'un éditeur de texte (\verb|emacs|, \verb|gedit|,... mais pas \textbf{Word}), créer un fichier texte \verb|rob.txt|.

Par exemple entrer la commande : \verb|emacs rob.txt|
\else
\item A l'aide d'un éditeur de texte, créer un fichier texte \verb|main.txt|.

Par exemple entrer la commande : \verb|emacs main.txt|
\fi
\item Ecrire quelques mots dans le fichier et sauvegarder.
\item Vérifier que vous n'avez plus accès au terminal d'où est lancé l'éditeur de texte tant que vous n'avez pas quitté l'éditeur. Pour éviter cela, on peut lancer les applications au second plan ("background") en ajoutant le symbole \verb|&| à la fin d'une commande.
\ifdefined\matlab

Exemple : \verb|emacs rob.txt &|
\else

Exemple  : \verb|emacs main.txt &|
\fi
\item Quitter l'éditeur de texte et le relancer comme indiqué précédemment. Vérifier que vous retrouvez bien le texte entré dans le fichier et que, maintenant, vous avez accès au terminal.
\item Afficher le contenu du fichier texte dans le terminal directement via la commande :

\verb|cat|~\textit{nom\_du\_fichier.txt}
\end{enumerate}

\exercice{- Connexion à distance}
Procédure : 
\begin{enumerate}
\item ouvrir un Shell.
\item Se connecter sur une autre machine :

\verb|ssh -Y pc????| où \verb|pc????| désigne le nom d'une machine de la salle de TP (par exemple pc4021)
\item Vérifier que vous êtes toujours dans votre répertoire personnel et que vous avez accès à vos fichiers.
\item Avec la commande \texttt{who}, observez quels sont les utilisateurs connectés sur la même machine que vous.
\item Observer les processus sur la machine locale et la machine distante. Pour cela, utilisez la commande \texttt{ps} (et \texttt{top} pour voir les processus actifs).
\end{enumerate}
Vous pouvez travailler depuis chez vous, en vous connectant sur le \texttt{pc57} via la commande (dans un shell Linux) :

\verb|ssh -Y login@pc57.polytech.upmc.fr|

Dans l'hypothèse (peu probable) où votre système d'exploitation serait \textit{Windows}, il y a un outil qui est décrit dans Moodle (MobaXterm, fichier \verb|Travailler_sous_windows.pdf|).

Notez que le \texttt{pc57} n'est que le point d'entrée dans le réseau polytech. Il est impossible d'y travailler. Une fois que l'on est connecté sur \texttt{pc57}, il faut se connecter via \verb|ssh| sur un autre PC (normalement, les PC "toujours" disponibles sont pc4021, pc4022, pc4023, pc4024, pc4025 et pc4026).
